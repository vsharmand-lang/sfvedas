          <h2>Test Your Understanding</h2>

          <div class="quiz-question" id="q1">
            <p class="quiz-prompt">1. Your team is migrating from Aura to LWC. They want to embed a new LWC component inside an existing Aura parent while the parent is migrated later. Is this allowed?</p>
            <div class="quiz-option" onclick="answer(this,'q1','right')">Yes — LWC components can be embedded inside Aura parents. This is the recommended bottom-up migration approach.</div>
            <div class="quiz-option" onclick="answer(this,'q1','wrong')">No — LWC and Aura components cannot coexist in the same component tree under any circumstances</div>
            <div class="quiz-option" onclick="answer(this,'q1','wrong')">Yes, but only if the Aura parent uses the c:container wrapper component provided by Salesforce</div>
          </div>

          <div class="quiz-question" id="q2">
            <p class="quiz-prompt">2. An architect argues that migrating stable, rarely-used Aura components should happen immediately to reduce technical debt. What is the most sound counter-argument?</p>
            <div class="quiz-option" onclick="answer(this,'q2','wrong')">Aura components will be automatically migrated by Salesforce in the next major release, so migration effort is wasted</div>
            <div class="quiz-option" onclick="answer(this,'q2','right')">Migration cost exceeds performance benefit for rarely-accessed components; prioritise by measured impact — high-traffic, frequently-rendered components deliver the greatest ROI</div>
            <div class="quiz-option" onclick="answer(this,'q2','wrong')">Aura is not actually in maintenance mode — it continues to receive new features alongside LWC</div>
          </div>

          <div class="quiz-question" id="q3">
            <p class="quiz-prompt">3. An LWC component on a high-traffic page needs to communicate with an Aura component in the same page layout. What mechanism should be used?</p>
            <div class="quiz-option" onclick="answer(this,'q3','wrong')">Aura Application Events — fire the event from LWC via Apex and consume it in the Aura component</div>
            <div class="quiz-option" onclick="answer(this,'q3','wrong')">Standard DOM custom events — dispatch from LWC and capture with an Aura event handler attribute</div>
            <div class="quiz-option" onclick="answer(this,'q3','right')">Lightning Message Service (LMS) — the framework-agnostic messaging channel that works for LWC-to-Aura, Aura-to-LWC, and LWC-to-LWC communication</div>
          </div>
